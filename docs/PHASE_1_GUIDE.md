# Phase 1: Foundation - Learning Guide

**Duration:** Weeks 1-2  
**Goal:** Understand data structures and how to store information

## What You'll Learn

### Week 1: Data Collection & Processing
- How to structure data from real sources
- How to parse and clean information
- How to think about data organization

### Week 2: Database Setup
- How to use SQLite (simple, local database)
- How to design tables for data
- How to query and retrieve information

---

## Why This Order?

Every AI system needs three things:
1. **Data In** - Collect and clean data
2. **Data Storage** - Store it somewhere reliable
3. **Data Out** - Use it to make decisions

You're learning #1 and #2 first. This is the foundation everything else depends on.

## Real-World Context

### Why Data Processing?

Imagine:
- A sales client wants to understand their competitors' websites
- You use a tool (SEO Frog) to audit that website
- That audit creates thousands of data points
- You need to:
  - Understand what data you have ✓ (Task 1-2)
  - Store it reliably ✓ (Task 3)
  - Later: Use it to make recommendations (Phase 2)

### Why SQLite?

- Simple (no server setup needed)
- Fast enough for learning
- Same concepts work for bigger databases later
- You'll use this in real projects

## Key Concepts to Understand

### Data Structures
- **Lists** - Ordered collection of items
- **Dictionaries** - Key-value pairs (like a map)
- **DataFrames** - Tables of data (like Excel)
- **JSON** - Format for storing structured data

### Database Basics
- **Table** - Like an Excel spreadsheet
- **Column** - A category of data
- **Row** - One record of data
- **Query** - A question you ask the database

## Common Questions

**Q: I don't understand something in an Issue**  
A: That's normal! Look at the example code provided. If still stuck, break it down into smaller pieces.

A: Check the error message. Google it. Leave a comment on the GitHub Issue. We learn from breaking things.

**Q: How long should this take?**  
A: 4-8 hours total spread over 1-2 weeks. Don't rush. Understand each piece.

**Q: Do I need to memorize everything?**  
A: No. You need to understand the concepts. Code you can look up.
